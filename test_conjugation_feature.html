Tensor Conjugation Feature

The conjugation feature allows you to transform a tensor by swapping its in and out legs while toggling a prime (') in its name.

What Conjugation Does:

  • Name Change: Toggles prime (') in the tensor name (adds if absent, removes if present)
  • Leg Relabeling: Swaps in and out leg numbers
  • Separator Update: Updates the leg separator accordingly
  • Visual Update: Colors update to reflect the new leg types
  • Reversible: Applying conjugation twice returns to the original state

Conjugation Rules:

Given: Tensor with n out-legs and m in-legs (separator = n)
After conjugation:
  • Old out-legs (1...n) → New in-legs (m+1...m+n)
  • Old in-legs (n+1...n+m) → New out-legs (1...m)
  • New separator = m (number of old in-legs)
  • Name toggles prime: "T" → "T'" or "T'" → "T"

Example: 4-Leg Tensor with Separator = 2

Before Conjugation:
• Name: "T"
• Separator: 2
• Leg 1: Out-leg (teal)
• Leg 2: Out-leg (teal)
• Leg 3: In-leg (red)
• Leg 4: In-leg (red)
After Conjugation:
• Name: "T'"
• Separator: 2
• Leg 1: Out-leg (teal)
• Leg 2: Out-leg (teal)
• Leg 3: In-leg (red)
• Leg 4: In-leg (red)

Name Generation:

Automatic Name Assignment:
• If tensor has no name → Assigns "T1", "T2", etc.
• If tensor has name without prime → Adds prime (e.g., "A" → "A'")
• If tensor has name with prime → Removes prime (e.g., "A'" → "A")
• Conjugation is reversible: "A" ↔ "A'"

How to Use:

  1. Create a tensor with some legs classified as in/out
  2. Right-click on the tensor to open context menu
  3. Click "Conjugate Tensor" button
  4. Observe changes:
    • Name toggles prime symbol
    • Leg numbers are relabeled
    • Colors update to show new in/out classification
    • Separator value updates

Mathematical Interpretation:

  • Tensor Conjugation: Represents the adjoint/transpose operation
  • Leg Swapping: Exchanges input and output indices
  • Prime Notation: Standard notation for conjugated tensors
  • Contraction Rules: Maintained - still only in→out connections allowed

Technical Implementation:

  • conjugateTensor() method handles the transformation
  • Global nextTensorNumber counter ensures unique names
  • Leg mapping algorithm preserves all leg properties
  • Context menu updates automatically after conjugation
  • Save/load functionality preserves conjugated state

Use Cases:

  • Quantum Circuits: Creating adjoint operations
  • Tensor Networks: Building complex contractions
  • Matrix Operations: Representing transposed matrices
  • Physical Systems: Modeling time-reversed processes
